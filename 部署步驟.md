# 部署步驟

## 1. 解壓縮

不要直接把 ZIP 放進 GitHub。請先解壓縮，再把資料夾內的檔案放到 GitHub 專案第一層。

GitHub 第一層應該看到：

```text
index.html
package.json
package-lock.json
README.md
部署步驟.md
```

## 2. 推上 GitHub

```bash
git add .
git commit -m "add ARC V13 one click payment selection"
git push origin main
```

## 3. Vercel 設定

建議使用：

```text
Framework Preset：Vite
Root Directory：./
Build Command：npm run build
Output Directory：dist
Install Command：npm install --no-audit --no-fund --legacy-peer-deps
Development Command：npm run dev
```

若使用 pnpm：

```text
Build Command：pnpm run build
Install Command：pnpm install --frozen-lockfile=false
Development Command：pnpm run dev
```

## 5. 取消繳費規則

```text
選案繳費頁面 → 未繳費案件 → 取消繳費
系統會要求輸入取消原因
取消後案件保留，不會刪除
取消後不進待繳、不進待領
取消後金額不計入統計
統計頁會列入取消案件
操作紀錄會留下取消人員、時間與原因
```

## 6. 權限

```text
管理員：全部權限
行政：不可操作仲介與扣款帳號、系統設定、會計對帳、會計查詢；不可刪除批次
會計：只有會計對帳、會計查詢，可調整帳戶餘額；不可刪除批次
```

## 7. 更新後若瀏覽器仍看到舊資料

這版使用 localStorage。若瀏覽器已有舊版暫存資料，可能還會看到舊資料。可用管理員登入後進入系統設定重置 Demo 資料，或清除瀏覽器 localStorage。

## 8. 正式上線提醒

目前為 V13 單檔樣式版，帳號、綁定個人與資料存在瀏覽器 localStorage。若需要多人跨電腦共用同一份帳號、資料與操作紀錄，下一階段需改接 Supabase Auth + Supabase Database。

## 本版編號規則

案件編號：ARC仲介代碼西元年月日001，以此類推。
範例：ARCFW20260630001、ARCWC20260630001。

批次編號：仲介代碼西元年月日001，以此類推。
範例：FW20260630001、WC20260630001。


## 本版提醒

- 待繳案件會依仲介別分開顯示。
- 不同仲介不可同一批繳費，但不同仲介可以一起傳真與領件。
- 單筆 / 批次送件已刪除行政分類欄位。

## 本版新增

- 選擇要繳費案件可搜尋雇主、工人名、團號、案件編號。
- 選擇要繳費案件表格新增團號欄位。
- 待繳案件區塊改成可滑動查看，避免案件量多時頁面過長。
- 最近繳費批次預設只顯示待財務確認，可篩選打開財務已確認或全部批次。
