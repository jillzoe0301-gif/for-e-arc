# ARC V13 正式版部署步驟

## 1. Supabase 建立資料表與後端函式

打開 Supabase → SQL Editor，執行：

```text
supabase/001_formal_schema_rpc.sql
```

這個 SQL 會建立：

```text
arc_users
arc_sessions
arc_app_state
arc_login
arc_get_state
arc_save_state
arc_list_users
arc_admin_create_user
arc_admin_update_user
arc_admin_set_user_active
arc_admin_delete_user
arc_logout
```

## 2. Vercel 環境變數

到 Vercel 專案 → Settings → Environment Variables，新增：

```text
VITE_SUPABASE_URL=https://你的-project-ref.supabase.co
VITE_SUPABASE_ANON_KEY=你的 Supabase anon public key
```

不要放 service_role key。

## 3. GitHub 第一層檔案

不要直接把 ZIP 放進 GitHub。請先解壓縮，然後把資料夾內檔案放到 GitHub 專案第一層。

第一層應該看到：

```text
index.html
package.json
package-lock.json
README.md
部署步驟.md
.env.example
supabase/001_formal_schema_rpc.sql
```

## 4. 推上 GitHub

```bash
git add .
git commit -m "release ARC V13 formal Supabase version"
git push origin main
```

## 5. Vercel 設定

```text
Framework Preset：Vite
Root Directory：./
Build Command：npm run build
Output Directory：dist
Install Command：npm install --no-audit --no-fund --legacy-peer-deps
Development Command：npm run dev
```

## 7. 上線後確認

登入後請確認：

```text
1. 右上角顯示登入者與角色
2. 管理員看得到全部選單
3. 行政看不到會計對帳、會計查詢、仲介與扣款帳號、系統設定
4. 會計只看得到會計對帳與會計查詢
5. 選案繳費不同仲介不可同批
6. 新增或修改資料後，換另一台電腦登入也看得到
```

本版為 V13 正式版，會連線 Supabase 共用資料。若沒有設定 Supabase 環境變數，正式版會阻擋登入。
