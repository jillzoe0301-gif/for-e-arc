# ARC V13 正式版｜重入境許可直接歸入案件查詢版

本版本延續 ARC V13 正式版 Supabase 架構與 V13 版面樣式，新增本次需求：

## 本版更新

- 申請項目新增「重入境許可」。
- 重入境許可送件後會直接歸入「案件查詢」。
- 重入境許可不會進入「待繳案件」。
- 重入境許可不會進入「待領案件」。
- 單筆送件選擇重入境許可後，儲存完成會自動切到案件查詢頁。
- 批次送件若整批都是重入境許可，儲存完成會自動切到案件查詢頁。
- 批次送件若有混合項目，重入境許可會自動歸入案件查詢，其餘案件依原流程進待繳或待領。
- 重入境許可不需要輸入金額，金額不列入申請金額與繳款統計。
- 案件查詢會顯示狀態為「案件查詢留存 / 不需繳費、不需領件」。
- 既有遠端或本機資料若尚未有「重入境許可」項目，系統會自動補入申請項目清單。

## 保留規則

- 取消申請不進待繳與待領。
- 不同仲介可以同批批次送件。
- 不同仲介不可同一批繳費。
- 不同仲介可以一起傳真與領件。
- 金額欄位維持 500 級距上下調整。
- Excel 垂直貼上功能保留。

## 部署

解壓縮後，將檔案放到 GitHub 專案第一層，再推送到 main。

```bash
git add .
git commit -m "add ARC V13 reentry permit direct case query flow"
git push origin main
```

本次為前端流程與畫面調整，不需要重新執行 Supabase SQL。
