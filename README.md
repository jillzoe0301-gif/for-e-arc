# ARC V13 正式版｜自動登入、批次 5 列、預計領件區與傳真領件紀錄

本版維持 V13 內容、呈現方式、樣式、規則與 icon 基準，只調整本次指定項目。

## 本版新增 / 修正

### 登入頁面
- 新增「下次打開直接登入」勾選框。
- 勾選後下次開啟會自動進入上次登入帳號畫面。
- 登出會清除自動登入狀態。
- 不明文儲存密碼；正式版只保留 session token、登入者 Email / 名稱 / 角色及自動登入狀態。

### 批次送件
- 「增加列」改為每次新增 5 列。
- 新增列完整包含既有批次欄位。
- 批次送出仍逐列判斷有效資料並全部儲存。

### 傳真領件
- 傳真領件表於承辦後方新增「收據順序」欄位。
- 收據順序可手動輸入數字並儲存於明細。
- 勾選案件時需同時具備：收件編號、外字末五碼、收據順序。
- 符合條件後進入「預計領件區」。
- 取消勾選或清空必要欄位會自動移出預計領件區。
- 預計領件日預設以每週四為基準：週一含以前為當週四，週二起為下週四。
- 預計領件區可切換領件日查看。
- 產生傳真領件單時建立「傳真領件紀錄」，紀錄編號格式為 `PYYYYMMDD-01`。
- 紀錄明細可查看，每筆可標記「本次未領到」。
- 本次未領到會保留原紀錄，並讓案件回到預計領件區，領件日預設下一個可領件日。
- 列印傳真領件單時會印出領件日期、紀錄編號、收件編號、外字末五碼、收據順序等欄位。

## 部署提醒

本次為前端功能與 app_state JSON 資料結構延伸，不需要重新跑 Supabase SQL。

正式部署仍需在 Vercel 設定：

```text
VITE_SUPABASE_URL
VITE_SUPABASE_ANON_KEY
```
