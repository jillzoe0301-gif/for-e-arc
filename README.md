# ARC 居留證控管系統｜V13 權限版＋取消繳費紀錄

本版本以 Demo V13 的內容、呈現方式、樣式、規則與 icon 為基準製作，並加入 Email 登入、角色權限、帳號管理、帳號綁定個人、會計調整餘額、操作紀錄、資料救回、取消申請、取消繳費紀錄與申請項目維護。


## 本版新增：繳費依仲介分開

- 「選擇要繳費案件」已依仲介別分開顯示。
- 不同仲介不可同一批繳費；若已勾選某仲介案件，再勾選其他仲介會被阻擋。
- 扣款帳號會依本批次仲介自動篩選。
- 不同仲介仍可一起傳真與領件。

## 本版調整：送件頁刪除行政分類

- 單筆送件與批次送件畫面已刪除「行政分類」欄位。
- 系統內部簽收與統計所需分組會自動以「承辦」帶入，不需再另外填寫行政分類。

## 本版新增：取消繳費

- 在「選案繳費」頁面，未繳費的送件案件可點選「取消繳費」。
- 點選後會跳出取消原因，原因不可空白。
- 取消後案件仍會保留，不會刪除。
- 取消繳費案件不會再進入待繳案件，也不會進入待領案件。
- 取消繳費案件的原金額會保留作參考，但不會計入申請金額統計或繳款總額。
- 案件統計頁會顯示在「取消案件」區塊。
- 操作紀錄會記錄取消人員、取消時間與取消原因。

## 權限規則

- 管理員：全部權限。
- 行政：可操作送件、繳費、傳真領件、案件查詢、統計、匯出、聯絡資訊、操作紀錄；不可操作仲介與扣款帳號、系統設定、會計對帳、會計查詢。
- 會計：只顯示會計對帳、會計查詢，並可在會計對帳頁調整帳戶餘額。
- 刪除繳費批次：只開放管理員，行政與會計不可刪除批次。
- 被刪除的資料可由「操作紀錄 / 資料救回」救回。救回按鈕限管理員使用。

## 申請項目

已預先設定：

- 新入境初次（紙本）
- 新入境展延（卡式）
- 續聘展延
- 承接展延
- 換護照展延
- 報備不製證
- 遺失補發
- 資料異動
- 中階居留證
- 雙語居留證
- 取消申請

「取消申請」不會進入待繳案件，也不會進入待領案件。

管理員可在「系統設定 → 申請項目維護」新增、修改、刪除申請項目。

## 會計對帳

- 帳戶餘額放在會計對帳頁最上方。
- 會計與管理員可調整帳戶餘額，調整會留下操作紀錄。
- 預設只顯示「待財務確認」批次。
- 可用篩選切換：待財務確認、財務已確認、全部批次。

## 列印

- 傳真與簽收單列印為 A4 直式。
- 列印按鈕只列印指定表單區塊，不列印整個系統頁面。

## 操作紀錄 / 資料救回

- 新增、修改、刪除、取消繳費、會計確認、領件異動、登入、登出都會留下紀錄。
- 刪除採軟刪除，不會真正移除資料。
- 可救回案件、繳費批次、扣款帳號、行政選項、登入帳號。

## 重要提醒

這版為了完全保留 V13 的單檔樣式與呈現方式，帳號與資料會暫存在瀏覽器 localStorage。正式上線若要跨電腦共用帳號與資料，下一階段建議接 Supabase Auth + Supabase Database。

## 部署方式

1. 解壓縮本專案。
2. 將資料夾內檔案放到 GitHub 專案第一層。
3. Vercel 使用 Vite 部署。
4. Build Command 使用 `npm run build` 或 `pnpm run build`。

## 編號規則更新

- 案件編號：`ARC` + 仲介代碼 + 西元年月日 + 三碼流水號，例如 `ARCFW20260630001`。
- 批次編號：仲介代碼 + 西元年月日 + 三碼流水號，例如 `FW20260630001`。
- 系統會依同仲介、同日期自動遞增 001、002、003。
- 舊版 localStorage 測試資料會於第一次載入時自動轉換成新編號，並保留操作紀錄。
