# ARC V13 正式版｜傳真領件單筆領件、列印格式與會計查詢搜尋

本版維持 V13 內容、呈現方式、樣式、規則與 icon 基準，只調整本次指定的傳真領件與會計查詢項目。

## 本版新增 / 修正

### 傳真領件
- 預計領件區每筆明細新增「單筆領件」按鈕。
- 單筆領件只處理該筆案件，會建立獨立傳真領件紀錄並可直接列印傳真領件單與簽收單。
- 批次領件仍可一次處理目前領件日的預計領件案件。
- 移除畫面上的「領件狀況維護」區塊，領件狀態改由「單筆領件」與「本次未領到」按鈕處理。
- 傳真領件紀錄明細保留「本次未領到」按鈕。
- 本次未領到會保留原紀錄，並讓案件回到預計領件區，領件日預設下一個可領件日。

### 傳真領件單列印
- 傳真領件單欄位順序調整為：編號、收費日期、收件編號、IC 卡、張數、經手人後四碼、外字五碼、雇主、工人、承辦、收據順序。
- 傳真領件單下方新增固定資訊：仲介名稱、電話、承辦、總領件數。
- 收據順序會正確列印。

### 簽收單列印
- 簽收單列印欄位調整為：領件日、承辦、雇主、工人、團號、申請項目。
- 簽收單依承辦排序。
- 同一承辦案件集中於同一區塊。
- 每位承辦下方都有承辦簽名欄位與本承辦總領件數。

### 會計查詢
- 會計查詢新增「雇主 / 工人名稱查詢」欄位。
- 可用雇主名稱、雇主關鍵字、工人姓名、工人關鍵字查詢。
- 可與原本起訖日期、顯示方式、確認狀態篩選一起使用。
- 查無資料時顯示「查無符合資料」。

## 部署提醒

本次為前端功能與 app_state JSON 資料結構延伸，不需要重新跑 Supabase SQL。

正式部署仍需在 Vercel 設定：

```text
VITE_SUPABASE_URL
VITE_SUPABASE_ANON_KEY
```
