# ARC V13 正式版｜搜尋完整姓名與系統設定修正版

本版依第二十二項需求修正全系統搜尋 / 查詢 / 查找 / 篩選輸入與系統設定點選無反應問題，維持既有 V13 內容、呈現方式、樣式、權限、列印、統計與 Supabase 正式版架構。

## 本版修正重點

- 全系統搜尋比對改為統一標準化文字：trim、移除不可見字元、移除換行 / tab、全形半形統一、大小寫不敏感、連續空白合併。
- 搜尋完整雇主名稱、完整工人姓名、部分名稱、複製貼上名稱，結果會一致。
- 支援資料內含空白時搜尋不含空白的名稱，例如資料為「王 小明」，搜尋「王小明」也可比對。
- 搜尋比對改用 includes 包含比對，不只做完全相等比對。
- 付款搜尋、案件查詢、財務查詢、傳真/領件搜尋、聯絡資訊搜尋、操作紀錄、系統設定搜尋皆套用同一組搜尋規則。
- 強化中文注音 composition 狀態：組字中不執行搜尋、不清空輸入、不重設 input value，compositionend 後再套用搜尋。
- 角色判斷相容中文角色名稱：管理員 / 行政 / 會計，避免因角色文字不同導致系統設定無法進入。
- 點選無權限頁面會顯示提示；點選系統設定但無權限時提示：「您沒有進入系統設定的權限。」
- 系統設定頁面保留管理員可維護項目：申請項目、仲介、帳戶、承辦/人員、傳真/領件、提醒事項、列印設定。
- 頁面載入若發生錯誤，會顯示錯誤卡片，不會完全沒有反應。

## 套用方式

解壓縮後，將資料夾內檔案放到 GitHub 專案第一層，確認第一層可看到：

- index.html
- package.json
- package-lock.json
- README.md
- 部署步驟.md
- .env.example
- supabase/

執行：

```bash
git add .
git commit -m "fix ARC V13 search full name and settings page"
git push origin main
```

本次為前端搜尋、權限判斷與頁面渲染防呆修正，不需要重新執行 Supabase SQL。
