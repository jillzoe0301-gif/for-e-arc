# ARC V13 正式版｜申請項目流程分流修正版部署步驟

## 1. 解壓縮

請先解壓縮 zip，不要直接把 zip 上傳到 GitHub。

## 2. 覆蓋 GitHub 專案檔案

將解壓縮後的檔案放到 GitHub 專案第一層，第一層需看到：

```text
index.html
package.json
package-lock.json
README.md
部署步驟.md
.env.example
```

## 3. 推送 GitHub

```bash
git add .
git commit -m "fix ARC V13 application item workflow routing"
git push origin main
```

## 4. Vercel 自動部署

GitHub 推送後，Vercel 會自動重新部署。

## 5. Supabase

本次不用重新跑 Supabase SQL。

## 6. 驗收項目

1. 資料異動且金額 0 元：登記後直接進入傳真/領件，不出現在居留證繳費。
2. 資料異動且金額大於 0 元：維持一般繳費流程。
3. 重入境許可：登記後只在案件查詢留存，不進繳費、不進傳真/領件。
4. 新入境初次（紙本）：需繳費，繳費後案件查詢留存，不進傳真/領件。
5. 報備不製證：需繳費，繳費後案件查詢留存，不進傳真/領件。
6. 其他一般案件：流程不受影響。
