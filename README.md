# ARC V13 正式版｜申請項目流程分流修正版

本版依第二十一項需求修正案件流向規則，維持既有 V13 內容、呈現方式、樣式、權限、列印、統計與 Supabase 正式版架構。

## 本版修正重點

- 資料異動且金額 0 元：登記後直接進入傳真/領件，不進入居留證繳費。
- 重入境許可：登記後只保留於案件查詢，不進入居留證繳費，也不進入傳真/領件。
- 新入境初次（紙本）：登記後進入居留證繳費；繳費完成後直接案件查詢留存，不進入傳真/領件。
- 報備不製證：登記後進入居留證繳費；繳費完成後直接案件查詢留存，不進入傳真/領件。
- 其他一般案件：維持原流程，登記 → 繳費 → 傳真/領件 → 案件查詢留存。
- 已新增「重入境許可」為預設申請項目，預設金額 0 元。
- 案件查詢、匯出與狀態文字同步顯示：免繳費、待傳真/領件、已登記 / 查詢留存、已繳費 / 查詢留存。

## 套用方式

解壓縮後，將資料夾內檔案放到 GitHub 專案第一層，確認第一層可看到：

- index.html
- package.json
- package-lock.json
- README.md
- 部署步驟.md
- .env.example

執行：

```bash
git add .
git commit -m "fix ARC V13 application item workflow routing"
git push origin main
```

本次為前端流程與 app_state JSON 判斷邏輯修正，不需要重新執行 Supabase SQL。
