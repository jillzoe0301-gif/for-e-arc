# 部署步驟

## 1. 解壓縮

不要直接把 ZIP 放進 GitHub。請先解壓縮，再把資料夾內的檔案放到 GitHub 專案第一層。

GitHub 第一層應該看到：

```text
index.html
package.json
package-lock.json
README.md
部署步驟.md
```

## 2. 推上 GitHub

```bash
git add .
git commit -m "update ARC V13 cancellation items accounting filter and default accounts"
git push origin main
```

## 3. Vercel 設定

建議使用：

```text
Framework Preset：Vite
Root Directory：./
Build Command：npm run build
Output Directory：dist
Install Command：npm install --no-audit --no-fund --legacy-peer-deps
Development Command：npm run dev
```

若使用 pnpm：

```text
Build Command：pnpm run build
Install Command：pnpm install --frozen-lockfile=false
Development Command：pnpm run dev
```

## 5. 權限

```text
管理員：全部權限
行政：不可操作仲介與扣款帳號、系統設定、會計對帳、會計查詢；不可刪除批次
會計：只有會計對帳、會計查詢，可調整帳戶餘額；不可刪除批次
```

## 6. 更新後若瀏覽器仍看到舊帳號

這版會自動套用新的帳號、申請項目與行政清單。若瀏覽器仍顯示舊資料，可在登入後由管理員進入系統設定重置 Demo 資料，或清除瀏覽器 localStorage。

## 7. 正式上線提醒

目前為 V13 單檔樣式版，帳號、綁定個人與資料存在瀏覽器 localStorage。若需要多人跨電腦共用同一份帳號、資料與操作紀錄，下一階段需改接 Supabase Auth + Supabase Database。
