# ARC 居留證控管系統｜V13 完全重製版 + 登入權限 + 操作紀錄 / 資料救回

本版本以 Demo V13 的內容、呈現方式、樣式、規則與 icon 為基準重新製作，並加入 Email 登入、角色權限、帳號管理、操作紀錄與資料救回。

## 預設登入帳號

| 角色 | Email | 密碼 | 權限 |
|---|---|---|---|
| 管理員 | admin@forward.local | admin123 | 全部功能 |
| 行政 | arc@forward.local | arc123 | 可操作送件、繳費、傳真領件、查詢、統計、匯出、聯絡資訊、操作紀錄；不可操作仲介與扣款帳號、系統設定、會計對帳、會計查詢 |
| 會計 | accounting@forward.local | acc123 | 只有會計對帳、會計查詢 |

## 權限規則

- 管理員：全部權限。
- 行政：不能調整仲介與扣款帳號、不能操作系統設定、不能操作會計對帳與會計查詢。
- 會計：只顯示與操作會計對帳、會計查詢。
- 被刪除的資料可由「操作紀錄 / 資料救回」救回。救回按鈕限管理員使用。

## 帳號後臺

管理員登入後進入「系統設定」，可操作：

- 新增帳號
- 修改 Email
- 修改姓名
- 修改角色
- 修改密碼
- 停用 / 啟用帳號
- 刪除帳號
- 從操作紀錄救回被刪除帳號

## 操作紀錄 / 資料救回

- 新增、修改、刪除、會計確認、領件異動、登入、登出都會留下紀錄。
- 刪除採軟刪除，不會真正移除資料。
- 可救回案件、繳費批次、扣款帳號、行政選項、登入帳號。

## 重要提醒

這版為了完全保留 V13 的單檔樣式與呈現方式，帳號與資料會暫存在瀏覽器 localStorage。正式上線若要跨電腦共用帳號與資料，建議下一階段接 Supabase Auth + Supabase Database。

## 部署方式

1. 解壓縮本專案。
2. 將資料夾內檔案放到 GitHub 專案第一層。
3. Vercel 使用 Vite 部署。
4. Build Command 使用 `npm run build` 或 `pnpm run build`。
